# Verification Codes Testing Guide - UI Flow

## Overview

This guide covers the **Pickup Codes** and **Delivery Codes** verification system. These secure 6-digit codes ensure that packages are handed off to the right person at pickup and delivery.

---

## Why Verification Codes?

**Security Benefits:**
- Prevents unauthorized pickups
- Confirms in-person handoff
- Protects both shipper and carrier
- Creates audit trail for disputes

**How It Works:**
- Codes are 6 random digits (e.g., "482915")
- Generated by the shipper/recipient
- Entered by the carrier to confirm handoff
- Codes expire after 30 minutes
- One-time use only

## Part 10: What to Verify After Each Test

### After Generating Code
- [ ] 6-digit code is displayed
- [ ] Code is visible in large font
- [ ] Timer shows ~30 minutes
- [ ] "Copy Code" button works
- [ ] Success message mentions chat
- [ ] Code appears in chat conversation

### After Entering Code
- [ ] Sheet closes on success
- [ ] Booking status updates
- [ ] Error shows for wrong code
- [ ] Can retry after error
- [ ] Carrier can't re-enter used code

### After Code Expires
- [ ] Timer reaches 0:00
- [ ] "Generate New Code" button appears
- [ ] Old code no longer works
- [ ] New code has fresh 30-minute timer

## Part 11: Common Issues & Solutions

| Issue | Cause | Solution |
|-------|-------|----------|
| No code button visible | Wrong booking status | Check if booking is "Confirmed" (pickup) or "In Transit" (delivery) |
| Button disabled | Code not generated yet | Ask shipper to generate code first |
| Code not working | Typo in entry | Double-check each digit |
| Code not working | Code expired | Ask shipper to generate new code |
| Code not in chat | Network delay | Pull to refresh chat |
| Timer showing 0:00 | Code expired | Generate new code |
| "Already used" error | Code was entered before | Cannot reuse - check booking status |

## Part 12: Prerequisites for Testing

### Shipper Account Needs
- [ ] Logged in as shipper
- [ ] Has a confirmed booking (for pickup code test)
- [ ] Has an in-transit booking (for delivery code test)

### Carrier Account Needs
- [ ] Logged in as carrier
- [ ] Has access to same bookings as shipper test account
- [ ] Second device or way to receive code

### Test Setup
- [ ] Two devices/accounts (shipper and carrier)
- [ ] Internet connection on both
- [ ] Booking in correct status for test type
